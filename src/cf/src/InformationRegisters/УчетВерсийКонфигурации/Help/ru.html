Подсистемы "Учет конфигураций" предназначена для возможности проведения обновления типовых измененных конфигураций. Возможность фиксации изменений при обновлении (после обновления) любых типовых конфигураций 1С, которые сняты с поддержки поставщика. При использовании данной подсистемы нет необходимости в фиксации изменений на бумажных носителях, в электронных блокнотах.

ИНСТРУКЦИЯ ПО РАБОТЕ С УЧЕТОМ ВЕРСИЙ КОНФИГУРАЦИЙ

1. В типовой конфигурации используется при внесении изменений методология "Купорос".
Основной принцип методологии состоит в следующем: вставка в шапке любого модуля определенного свода по изменениям. Данные вставки необходимо для удобства использовать из шаблонов текста 1С, входящих в поставку к конфигурации.

Для любого модуля объекта потребуется вставка вида:

//{{ *[Имя] [Фамилия] >>>>>>>>>>>>>>>>>>>>>>> ОПИСАНИЕ ИЗМЕНЕНИЙ >>>>>>>>>>>>>>>>>>>>>>>>>>>
//
// НОВЫЕ/ИЗМЕНЕННЫЕ МЕТАДАННЫЕ ОБЪЕКТА ОТСУТСТВУЮТ*
//  +[<ТипОбъекта>]<ДобавляемыйОбъект> - <Краткое описание изменений>
//  *[<ТипОбъекта>]<ИзменяемыйОбъект>   - <Краткое описание изменений>
//
// НОВЫЕ/ИЗМЕНЕННЫЕ ПРОЦЕДУРЫ И ФУНКЦИИ
// +[<Процедура/Функция>]<ДобавляемаяПроцедура> - <Краткое описание изменений>
// *[<Процедура/Функция>]<ИзменяемаяПроцедура>   - <Краткое описание изменений>
//
//}} *[Имя] [Фамилия]  <<<<<<<<<<<<<<<<<<<<<<< ОПИСАНИЕ ИЗМЕНЕНИЙ <<<<<<<<<<<<<<<<<<<<<<<<<<<


// Разделитель описания изменений модуля объекта >> *[Имя] [Фамилия] 
Перем ИзмененияМодуляОбъекта;

*Если в объектах метаданных есть изменения то необходимо убрать слово "ОТСУТСТВУЕТ"

Для любого модуля формы потребуется вставка:

//{{ *[Имя] [Фамилия] >>>>>>>>>>>>>>>>>>>>>>> ОПИСАНИЕ ИЗМЕНЕНИЙ >>>>>>>>>>>>>>>>>>>>>>>>>>>
//
// ФОРМА НЕ ИЗМЕНЯЛАСЬ*
// +[<ТипОбъекта>]<ДобавляемыйОбъект> - <Краткое описание изменений>
// *[<ТипОбъекта>]<ИзменяемыйОбъект> - <Краткое описание изменений>
//
// НОВЫЕ/ИЗМЕНЕННЫЕ ПРОЦЕДУРЫ И ФУНКЦИИ
// +[<Процедура/Функция>]<ДобавляемаяПроцедура> - <Краткое описание изменений>
// *[<Процедура/Функция>]<ИзменяемаяПроцедура> - <Краткое описание изменений>
//
//}} *[Имя] [Фамилия] <<<<<<<<<<<<<<<<<<<<<<< ОПИСАНИЕ ИЗМЕНЕНИЙ <<<<<<<<<<<<<<<<<<<<<<<<<<<

// Разделитель описания изменений модуля формы >> **[Имя] [Фамилия]
Перем ИзмененияМодуляФормы; 

Для общих модулей  используется вставка аналогичная для модулей объектов, только в ней в качестве разделителя используется функция - ИзмененияОбщегоМодуля().

*Если на самой форме есть измененения, то необходимо убрать слово "НЕ" во вставке
**Для типа объекта могут быть указаны различные типы объектов метаданных, например: Свойства, Роль, Реквизиты, Элементы и др.
***Для <Процедура/Функция> указывается либо "Процедура", либо "Функция" в зависимости от использования той или иной сущности

Шаблон.st можно сформировать из основной формы учета версий конфигурации (кнопка "Сформировать шаблон st для разработчика" ), для дальнейшего использования в измененных типовых конфигураций.

2. Выбираем в форме "Учет версий типовых конфигурация" необходимую конфигурацию и версию до которой необходимо обновить конфигурацию 1С.

2.1 По кнопке "Импорт из модулей" выполняем импорт данных из текста модулей (необходимо предварительно выгрузить из конфигурации все модуля).

2.2 По кнопке "Заполнить новую версию" автоматически произойдет заполнение на основание актуальных данных справочника объектов метаданных, обновленных в п.2.1. При автозаполнении, информация для таблицы изменений берется из справочника "Объекты метаданных конфигурации", закладка "Изменения по умолчанию".

3. Сохраним рабочую конфигурацию в файл, например work.cf. Для этого выберем пункт меню «Конфигурация» → «Сохранить конфигурацию в файл…». Производим сравнение и объединение конфигурации старой с новой в конфигураторе через Поддержку («Конфигурация» → «Поддержка» → «Обновить конфигурацию»). Планируемые изменения объектов метаданных фиксируем в регистре сведений "Учет версий конфигурации". Зафиксировав все изменения, которые необходимо будет произвести, устанавливается галочка "Проведен анализ изменений (еще до обновления конфигурации)".

4. После обновления конфигурации, проводим изменения с учетом отмеченных в пункте 3 данных объектов метаданных. Произведя изменения, галочкой "Выполнено изменение (после обновления конфигурации)", фиксируется факт окончательного завершения работы с объектом.

В результате работы, после обновления конфигурации не должно остаться данных без установленной галочки "Выполнено изменение (после обновления конфигурации)". Таким образом конфигурация полностью обновлена и все изменения модулей зафиксированы. Если в процессе обновления необходимо обновить информацию по изменениям (по умолчанию) в справочнике "Объекты метаданных конфигурации", то используется кнопка в форме записи "Записать изменения в ОМ" или "Загрузить изменения из ОМС" для обратной процедуры.
